# qualityFirst 参数使用指南

## 📌 概述

所有脚本（video2gif.sh、video2apng.sh、video2webp_stand.sh）现已支持 `qualityFirst` 参数开关，用于在**质量优先**和**平衡模式**之间切换。

---

## 🎯 两种模式对比

### 平衡模式（默认，qualityFirst=false）
- **目标**：压缩与质量平衡
- **特点**：文件大小适中，处理速度快
- **适用**：日常使用、网页展示、移动端
- **文件大小**：基准值

### 质量优先模式（qualityFirst=true/1）
- **目标**：极致色彩还原，肤色自然饱满
- **特点**：文件较大，处理时间长，细节完美
- **适用**：高端展示、人像特写、专业用途
- **文件大小**：约1.2-1.5倍基准值

---

| 参数 | 平衡模式 | 质量优先模式 | 效果 |
|------|---------|-----------|------|
| **降噪** | 0.3:0.2:0.8:0.6 | 0.2:0.1:0.5:0.3 | 质量优先保留更多细节 |

| **对比度** | 1.01-1.02 | 1.03 | 质量优先立体感更强 |
| **GIF阈值** | 90 | 95 | 质量优先保留更多色彩 |
| **APNG压缩** | 5 | 3 | 质量优先压缩更低 |
| **WebP质量** | 90 | 95 | 质量优先质量更高 |

## 🔍 质量优先模式的具体改进

### 1. 肤色饱满度提升

**平衡模式：**
```
eq=contrast=1.02:brightness=0.01:saturation=1.05
```

**质量优先模式：**
```
eq=contrast=1.03:brightness=0.01:saturation=1.12
```

**效果：**
- 饱和度提升 7%（1.05 → 1.12）
- 对比度提升 1%（1.02 → 1.03）
- 结果：肤色更生动、更自然、更有血色

### 2. 细节锐化强度提升

**平衡模式：**
```
unsharp=5:5:0.8:3:3:0.4
```

**质量优先模式：**
```
unsharp=7:7:1.2:4:4:0.6
```

**效果：**
- 锐化半径提升 40%（5 → 7）
- 锐化强度提升 50%（0.8 → 1.2）
- 结果：头发丝清晰、皮肤纹理细腻、毛孔可见

### 3. 降噪参数最小化

**平衡模式：**
```
hqdn3d=0.3:0.2:0.8:0.6
```

**质量优先模式：**
```
hqdn3d=0.2:0.1:0.5:0.3
```

**效果：**
- 降噪强度降低 33%
- 结果：保留更多肤色纹理、毛发细节、皮肤质感

### 4. 色彩聚类敏感度提升（GIF）

**平衡模式：**
```
palettegen=...threshold=90
```

**质量优先模式：**
```
palettegen=...threshold=95
```

**效果：**
- 阈值提升 5%（90 → 95）
- 结果：肤色渐变更平滑、色彩过渡更自然

## ⚙️ 参数说明

### GIF 脚本参数

```bash
./video2gif.sh <输入> <输出> <开始时间> <时长> [帧率] [宽度] [qualityFirst]
```

- `qualityFirst`：true/1 = 质量优先，false/0 或省略 = 平衡模式

### APNG 脚本参数

```bash
./video2apng.sh <输入> <输出> <开始时间> <时长> [帧率] [宽度] [压缩级别] [qualityFirst]
```

- `压缩级别`：0-9（质量优先模式下自动调整为3）
- `qualityFirst`：true/1 = 质量优先，false/0 或省略 = 平衡模式

### WebP 脚本参数

```bash
./video2webp_stand.sh <输入> <输出> <开始时间> <时长> [帧率] [宽度] [质量] [qualityFirst]
```

- `质量`：0-100（质量优先模式下自动调整为95）
- `qualityFirst`：true/1 = 质量优先，false/0 或省略 = 平衡模式

## 🔧 故障排查

### 问题：质量优先模式文件仍然很大

**原因**：分辨率或时长过长

**解决**：
```bash
# 降低分辨率
./video2gif.sh "input.mp4" "output.gif" 00:00:00 00:00:10 30 720 true

# 或缩短时长
./video2gif.sh "input.mp4" "output.gif" 00:00:00 00:00:05 30 1080 true
```

### 问题：质量优先模式处理时间太长

**原因**：高分辨率 + 强锐化 + 高质量

**解决**：
```bash
# 降低帧率
./video2apng.sh "input.mp4" "output.apng" 00:00:00 00:00:10 24 1080 3 true

# 或使用WebP（编码更快）
./video2webp_stand.sh "input.mp4" "output.webp" 00:00:00 00:00:10 30 1080 95 true
```
